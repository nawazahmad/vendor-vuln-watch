⚠ Running from file:// — live pulls will likely fail
This page was opened directly from disk (). Browsers send Origin: null on requests made from file:// pages, and NVD's API rejects that origin outright — every "Pull Latest" will fail instantly regardless of retries or an API key. This is not a bug in the tool, it's how the browser labels the request.

Fix: serve this file over HTTP instead. Quickest way — open a terminal in this file's folder and run python3 -m http.server 8000, then visit http://localhost:8000/vendor-vuln-watch.html. Once it's hosted on GitHub Pages (or any real domain), this warning disappears and pulls should work normally.
